---
title: Postgres Sink Configuration
---

A Postgres sink config defines how datagen connects and writes data to PostgreSQL.

### Example
```json
{
  "sink_name": "pluto_postgres",
  "sink_type": "postgres",
  "config": {
    "host": "localhost",
    "database": "datagen",
    "port": 5432,
    "username": "dg",
    "password": "dg",
    "batch_size": 1,
    "throttle": "10ms",
    "timeout": "30s"
  }
}
```

### Config fields

<div class="cli-flags-table equal-4">


| Field       | Type    | Required | Description                                  | Default |
|-------------|---------|----------|----------------------------------------------|---------|
| host        | string  | Yes      | PostgreSQL server hostname or IP             | -       |
| database    | string  | Yes      | Database name to write into                  | -       |
| port        | number  | No       | PostgreSQL port                              | 5432    |
| username    | string  | Yes      | Database user                                | -       |
| password    | string  | Yes      | Database password                            | -       |
| batch_size  | number  | No       | Records per batch insert                     | 1       |
| throttle    | string  | No       | Delay between batches (e.g., "10ms", "1s")   | -       |
| timeout     | string  | No       | Connection timeout (e.g., "30s", "1m")       | -       |

</div>

**Notes:**
- Postgres sink is case sensitive; table and column names must match exactly as defined in the database
- Ensure user has INSERT privileges on target tables
- Use appropriate `batch_size` and `throttle` to control load rate
- When `clear_data` is enabled, the target table is truncated using `TRUNCATE TABLE ... RESTART IDENTITY CASCADE`, which resets sequences and cascades to dependent tables
- Connection uses `sslmode=disable` by default
